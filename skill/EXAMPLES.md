# HARP examples

## Example 1 — KERMT (chemistry MAE optimisation)

The first end-to-end use of HARP. Target repo: a graph-neural-network
model predicting Rf values from molecular structure. The user asked
HARP to drive the Rf prediction MAE below 0.05 (baseline 0.0662)
without manual intervention.

### What the user did manually

```bash
git clone git@github.com:TAOYUZHOU/harness-auto-research-pipeline.git
cd harness-auto-research-pipeline
ln -s "$(pwd)/skill" ~/.cursor/skills/harp     # install skill

# Bootstrap interactive
bash skill/scripts/harp_init.sh
# Q1) Target repo:        /root/autodl-tmp/taoyuzhou/KERMT
# Q2) Workspace:          /root/autodl-tmp/taoyuzhou/harp-workspace (default ok)
# Q3) Train script:       tlc/scripts/train_c_v3_v4.py
# Q4) Metric:             best_val_mae, lt
# Q4)) Baseline / threshold: 0.0662 / 0.05
```

`harp_init.sh` then automatically:

1. Rendered `meta_info/project.yaml` from the template
2. Ran `init_workspace.sh` → created `harp-workspace/` with all files
3. Ran `quickstart.sh` → preflight (agent indexed the codebase via
   GitNexus MCP, recorded constitution hash)
4. Ran `install_cron.sh install` → cron line every 15 min

### What ran autonomously after that

| When | Engine action | Token cost (cumulative) |
|------|---------------|-------------------------|
| t=0 | preflight propose: agent registered first plan `p_v4d2_freeze_encoder_generalization`, committed config to KERMT, started `nohup` training | ~30 k |
| t=15 m | tick: no new logs, no plateau → no agent call | 0 |
| t=4 h | tick: in-flight scan shows epoch 67, best 0.1038, plateau=0 → still healthy, no agent call | 0 |
| t=4.5 h | polish_daemon: log.md changed (in-flight snapshot row added) → fresh chat, ~2 k tokens for zh log | +2 k |
| t=5.5 h | tick: training crossed plateau ≥ 10 → enters monitor mode, agent decides `git_discard` and starts next plan | +18 k |

### Lessons that came out of this run (auto-captured in `memory.md`)

1. `freeze_encoder=true` on the chemistry encoder hurt convergence —
   plateau early at 0.10 vs baseline 0.066.
2. `early_stop_epoch=50` was too patient for screening; subsequent
   plans use 20.
3. Cold-start propose mode worked: HARP started its first experiment
   without any human prompt past `harp_init.sh`.

## Example 2 — minimal target with no AGENT-EDITABLE blocks

If your training script doesn't have AGENT-EDITABLE markers, HARP
treats the *whole file* as editable. Recommended only for solo-author
research repos where you trust the agent to edit anywhere.

In `meta_info/project.yaml`:

```yaml
harness:
  targets:
    - editable_files:
        - train.py            # entire file is editable
```

Add to `userprompt.rules`:

```yaml
- >
  train.py has no AGENT-EDITABLE markers; the whole file is yours
  to modify, but never change the CLI signature
  (--config, --output) — the engine and other tools depend on it.
```

## Example 3 — multiple targets

HARP can drive optimisations on multiple repos in one workspace
(e.g. baseline + ablation in separate codebases):

```yaml
harness:
  targets:
    - name: model_a
      repo_path: /code/repo_a
      primary_metric: val_mae
      stop_threshold: 0.05
    - name: model_b
      repo_path: /code/repo_b
      primary_metric: val_mae
      stop_threshold: 0.05
```

The agent rotates between them; `log.md` lines carry `ANCHOR=` matching
each target's anchor namespace. `git keep / discard` happens in the
relevant target repo independently.
